Archives are organized by:
- **Feature** (filters, schema, etc.)
- **Type** (implementation, testing, fixes)
- **Status** (completed, fixed, done)

Use the filename convention to identify archive type:
- `*-implementation.md` - Feature implementation
- `*-tests.md` - Test documentation
- `*-complete.md` - Completed phases
- `*-fixed.md` - Bug fixes or issues resolved
- `*-summary.md` - Summary reports
- `*-status.md` - Status/progress documentation
